Strings

A Ruby string is a series of characters. A string can be created by invoking the String object, or using the literal notation:

					
						> string_1 = "This is a literal string"
						=> "This is a literal string"
						> string_2 = String::new("This is a class invoked string")
						=> "This is a class invoked string"

Strings in Ruby can be added together or "concatenated" to form new strings:

					
						> string_3 = string_1 + " and" + string_2
						=> "This is a literal string and This is a class invoked string"

Ruby strings are not limited to text characters, and can include numbers:

					
						> string_4 = "99 red balloons"
						=> "99 red balloons"

The to_s method can be used to convert non-string objects into strings, and the is_a?(String) method can be used to check whether an object is a string:

					
						> number_1 = 99
						=> 99
						> number_1.is_a?(String)
						=> False
						> string_5 = number_1.to_s
						=> "99"
						> string_5.is_a?(String)
						=> True

Integers

A Ruby Integer is a whole number. An integer is created using the literal notation:

					
						> integer_1 = 41
						=> 41
						> integer_2 = 14
						=> 14

Basic arithmetic operations can be performed on Ruby Integers:

					
						> integer_1 + integer_2
						=> 55
						> integer_1 - integer_2
						=> 27
						> integer_1 * integer_2
						=> 574
						> integer_1 / integer_2
						=> 2
						> integer_1 % integer_2
						=> 13

The to_i method can be used to convert non-integer objects into integers, and the is_a?(Integer) method can be used to check whether an object is an integer:

					
						> string_4 = "-9946"
						=> "-9946"
						> string_4.is_a?(Integer)
						=> False
						> integer_3 = string_4.to_i
						=> -9946
						> integer_3.is_a?(Integer)
						=> True

Arrays

A Ruby Array is a collection of objects that are ordered and indexed. An Array can be created using the literal notation or by invoking the Array object:

					
						> array_1 = ["string", 45, True, ["hide", "seek"], 0.546]
						=> ["string", 45, True, ["hide", "seek"], 0.546]
						> array_2 = Array.new(4, "Default")
						=> ["Default", "Default", "Default", "Default"]

Objects stored in arrays can be accessed by their index position. This is true of single and multi-dimensional arrays:

					
						> array_1[0]
						=> "string"
						> array_1[1]
						=> 45
						> array_1[-1]
						=> 0.546
						> array_1[3][1]
						=> "seek"

The unshift, shift, push, and pop methods can be used to destructively add or remove elements from the start or end of an array respectively: 

					
						> array_2 = [2,3,4]
						=> [2, 3, 4]
						> array_2.unshift(1)
						=> [1, 2, 3, 4]
						> array_2.push(5)
						=> [1, 2, 3, 4, 5]
						> array_2.shift
						=> 1
						> array_2
						=> [2, 3, 4, 5]
						> array_2.pop
						=> 5
						> array_2
						=> [2, 3, 4]

Hashes

Ruby Hashes are collections of key-value pairs. They are similar to Arrays except Arrays use integers to index elements and Hashes allow the use of any object type as the index (or key). A Hash can be created using the literal notation or by invoking a new instance of the Hash object:

					
						> hash_1 = {"Filmore" => 22, "Church" => "J", "Geary" => 38}
						=> {
							"Filmore" => 22,
						        "Church" => "J",
						          "Geary" => 38
						     }
						> hash_2 = Hash.new
						=> {}

Key-value pairs can be added to Hashes by defining the key and corresponding value. Similarly, the value of a key can be accessed using the key:

					
						> hash_1["Turk"] = 31
						=> 31
						> hash_1
						=> {
							"Filmore" => 22,
						        "Church" => "J",
						          "Geary" => 38,
						             "Turk" => 31
						     }
						> hash_1["Geary"]
						=> 38

The has_key? and has_value? methods can be used to check if a Hash has a certain key or value respectively:

					
						> hash_1.has_key?("Church")
						=> True
						> hash_1.has_key?("Fulton")
						=> False
						> hash_1.has_value?(22)
						=> True
						> hash_1.has_value?(21)
						=> False
