Ruby Hashes vs JavaScript Objects

A JavaScript Reflection

March 5, 2016

Introduction

As part of a crash-course into JavaScript this week, we learned about the JavaScript data structure that provides the same function as the Ruby hash. Called an "object," this JavaScript data structure is also set up and populated in a similar manner to Ruby.

Creating and Populating JavaScript Objects

Like a Ruby hash, JavaScript object is a list of property and associated value pairs, stored inside a pair of curly braces. The JavaScript object also allows for literal creation, and the adding of property/value pairs using syntax not too dissimilar than that used by Ruby as illustrated in the following example. 

            
            

            new_object = {};
            => {}
            new_object["property_1"] = "Hide";
            => "Hide"
            new_object["property_2"] = "Seek";
            => "Seek"
            new_object
            => {property_1: "Hide", property_2: "Seek"}
            new_object.property_1
            => "Hide"
            new_object["property_2"]
            => "Seek"

Accessing and Deleting JavaScript Objects

There are 2 methods in which a property/value pair can be accessed and/or changed in JavaScript. These methods are known as the "dot-notation" and the "bracket-notation" and are illustrated in the following example. 

            
            

            new_object.property_1
            => "Hide"
            new_object["property_2"]
            => "Seek"
            new_object.property_1 = "Sewing Machines"
            => "Sewing Machines"
            new_object.property_1
            => "Sewing Machines"
            new_object
            => {property_1: "Sewing Machines", "Seek"}

Notice how the bracket-notation method expects the property as a string even though the returned object displays the property without a pair of quotes. JavaScript will throw an error if the property name is not provided as a string when using the bracket notation, and there is no Ruby-symbol-like data structure that can be used in JavaScript to get around this rather peculiar behavior.

The Syntax used to delete a single property/value pair or a JavaScript object altogether is also slightly different than that used by Ruby: 

            
            

            delete new_object.property_1
            => true
            new_object
            => {property_2: "Sewing Machines"}
            delete new_object
            => true
            new_object
            => Error!

The delete new_object command in this case did not just get rid of the last remaining property/value pair (i.e. property_2: "Seek") leaving behind an empty object (i.e. {}), it deleted the object new_object entirely. This is why an error is returned subsequently when the new_object is passed as a command.

Conclusion

The need to store keys (or properties as JavaScript calls them) and corresponding values is a fairly universal coding requirement. The Python equivalent to a Ruby hash and a JavaScript object is called a dictionary and it allows for the same creation, population, access, and deletion functionality as do Ruby and JavaScript.

If JavaScript is the first language you are dabbling in, the object may seem a little peculiar and may take some getting used to in practice. Coming from a few weeks of dealing with hashes in Ruby, this was not the case for me, and so I encourage you to stick with it and you will surely see the power and usefulness of the object data structure. Good luck!
